在系统编程和 C 语言中，`exit(0)` 和 `exit(1)` 的本质区别在于它们向**父进程**或**操作系统**传递的“遗言”内容不同。这是一种约定俗成的通信协议。

---

### 1. 核心区别：成功与失败
虽然在语法上它们都会导致进程立即终止，但在语义上有着天壤之别：

* **`exit(0)`：表示“成功退出”**。
    * 它告诉系统：“程序已经圆满完成了任务，没有遇到任何错误。”
    * 在 `stdlib.h` 中，它对应宏 **`EXIT_SUCCESS`**。
* **`exit(1)`：表示“异常退出”**。
    * 它告诉系统：“程序出事了，任务没能正常完成。”
    * 在 `stdlib.h` 中，它对应宏 **`EXIT_FAILURE`**。
    * *注：非零值（如 1, 2, -1 等）通常都被视为不同程度或类型的错误。*

---

### 2. 谁在乎这个数字？
这个状态码主要发给两个对象：

#### A. 操作系统 / Shell
如果你在 Linux 终端运行程序，你可以通过查看特殊变量 **`$?`** 来获取上一个进程的退出状态：
```bash
./my_program
echo $?  # 如果输出 0，说明刚才运行成功；如果输出 1，说明报错了
```
这在编写 **Shell 脚本**（如 `&&` 或 `||` 逻辑）时至关重要。例如：`gcc main.c && ./a.out`。如果 `gcc` 返回了非 0 值（`exit(1)`），后面的 `./a.out` 就不会执行。

#### B. 父进程 (`waitpid`)
正如你之前关心的 `waitpid` 函数，父进程可以通过 `status` 参数拿到这个值。
* 父进程调用 `waitpid(pid, &status, 0)`。
* 使用宏 **`WEXITSTATUS(status)`** 提取。如果子进程写的是 `exit(1)`，这个宏拿到的就是 `1`。



---

### 3. 为什么不直接用 `return`？
在 `main` 函数里，`return 0` 等价于 `exit(0)`。但在**深层嵌套的函数**中，`return` 只能跳出当前函数，而 `exit` 可以直接终结整个进程并带走状态码。

### 总结对比

| 状态码 | 宏定义 | 含义 | 场景举例 |
| :--- | :--- | :--- | :--- |
| **0** | `EXIT_SUCCESS` | **成功** | 排序完成、计算结束、文件成功保存。 |
| **1** | `EXIT_FAILURE` | **通用错误** | 内存申请失败、文件找不到、输入参数错误。 |
| **>1** | 自定义 | **特定错误** | 比如 2 代表权限不足，3 代表超时（由程序员自定义）。 |

---

### 💡 一个小建议
在写代码时，为了让代码更具可读性，建议优先使用宏：
```cpp
#include <stdlib.h>

if (ptr == NULL) {
    perror("Malloc failed");
    exit(EXIT_FAILURE); // 比写 exit(1) 更专业
}
```
